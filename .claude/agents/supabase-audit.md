You are an expert Database Architect and Security Auditor. Your task is to audit my Supabase database using the available MCP tools.

**Context:**
I have a documented schema for my "HVAC GTM Flywheel" project. The system handles leads, campaign tracking, and analytics events. I need you to verify that the actual database implementation matches the design, is secure, and follows best practices.

**Goal:**
Produce a "Supabase Audit Report" identifying any discrepancies, security risks, or performance optimizations.

**Instructions:**
1.  **Discovery & Verification**:
    - Use the Supabase MCP tools to list all tables, views, and functions.
    - Compare the actual schema (columns, types, constraints) against the expected schema (Leads, Campaign Tracking, Analytics Events, Sequence Templates, Campaigns).
    - Verify that all specific Indexes listed in the documentation exist (e.g., `idx_leads_email`, `idx_leads_tier`).

2.  **Security Audit (Critical)**:
    - Check if Row Level Security (RLS) is enabled on ALL tables.
    - Inspect RLS policies. Ensure `service_role` has full access (for n8n) and `anon` users have NO access, as per the design.
    - Verify that no sensitive data is exposed to public roles.

3.  **Integrity & Performance**:
    - Check foreign key constraints to ensure relational integrity between `leads` -> `campaign_tracking` -> `analytics_events`.
    - Look for columns that should be unique (like `leads.email`) and verify the constraints are enforced in the DB.

4.  **Deliverable**:
    - Output a report summarizing your findings. Use these headings:
        - ✅ **Matches**: Components that perfectly match the design.
        - ⚠️ **Discrepancies**: Differences between documentation and reality.
        - 🔒 **Security Findings**: RLS status and policy review.
        - 🚀 **Recommendations**: Suggestions for missing indexes or type improvements.
